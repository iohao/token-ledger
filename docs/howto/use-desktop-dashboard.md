# 桌面看板使用指南

这份指南面向第一次使用 `TokenLedger` 桌面应用的用户，说明如何完成首次同步、查看数据和调整本地数据库路径。

适用对象：已经拿到桌面应用，或由开发者代为打包好应用的使用者。

如果你需要从源码启动或打包应用，请先看 [README](../../README.md)。

## 开始前准备

在打开应用前，先确认两件事：

1. 你的 Codex 会话文件存在于默认目录，或你已经设置了 `CODEX_HOME`。
2. 你允许应用读取本机上的 Codex 数据目录。

默认目录如下：

- macOS / Linux：`~/.codex`
- Windows：`%USERPROFILE%\\.codex`

应用会从 `CODEX_HOME/sessions/*.jsonl` 读取会话数据，并默认把统计结果写入：

- `CODEX_HOME/.codex-usage/usage.sqlite`

如果你已经通过环境变量 `CODEX_USAGE_DATABASE` 指定了数据库路径，应用会直接使用那个路径，而且界面里不能改写。

## 第一次打开后怎么做

第一次使用时，按下面顺序操作即可：

1. 打开应用，进入默认的“概览”页。
2. 先看顶部状态卡。
   如果显示“尚未同步”或存在待同步会话，继续下一步。
3. 点击“同步数据”。
4. 等待“同步进度”卡从准备、扫描、处理推进到完成。
5. 同步结束后，概览页会显示今日、7 日和本月统计。

同步完成后，你通常会看到这些信息：

- 最近同步时间
- 已追踪会话数
- 待同步会话数
- 最近 7 天的每日汇总

如果你正在同时使用 Codex，少量“待同步”会话可能会继续出现。这通常是正常现象，因为活跃 session 文件仍在写入。

## 查看各个页面

左侧栏提供 5 个主要页面，也支持快捷键切换：

- `Cmd/Ctrl + 1`：概览
- `Cmd/Ctrl + 2`：每月用量明细
- `Cmd/Ctrl + 3`：用量汇总
- `Cmd/Ctrl + 4`：按日查询用量
- `Cmd/Ctrl + 5`：关于

### 概览

适合快速看整体趋势，包括：

- 当前同步状态
- 最近同步时间
- 今日、最近 7 日、本月统计
- 最近 7 日汇总表

### 按日查询用量

适合按日期范围查看每天的数据。

使用方式：

1. 进入“按日查询用量”页。
2. 选择开始日期和结束日期。
3. 点击“查询按日用量”。

当前限制和表现：

- 单次最多查询 93 天
- 结果会保留空白日期，方便看波动
- 结果按页展示，每页 31 天

如果你选的范围超过 93 天，界面会直接提示缩小范围后再查。

### 用量汇总

适合看长期趋势。这里会按月份显示聚合数据，通常不需要额外操作。

### 每月用量明细

适合查看某个月内每天的用量明细。

使用方式：

1. 进入“每月用量明细”页。
2. 先选择年份。
3. 点击 12 个月按钮中的任意一个月份。
4. 界面会直接显示该月每天的模型用量明细表。

这个页面会复用按日明细表格的展示方式，但查询范围固定为单个月份，所以通常不需要再手动输入日期。

### 关于

这个页面主要用来确认本地路径和同步状态，包括：

- Codex 目录
- 源码仓库
- SQLite 数据库路径
- 解析版本
- 覆盖到的时间
- 扫描文件数
- 受影响会话数

## 修改 SQLite 路径

如果你想把统计数据库放到别的位置，可以在“关于”页修改。

操作步骤：

1. 打开“关于”页。
2. 在“路径设置”区域找到 `SQLite 路径` 输入框。
3. 输入新的完整路径。
4. 点击“保存路径”。

注意事项：

- 修改后，应用会切换到新的 SQLite 文件。
- 如果目标文件不存在，应用会在需要时创建它。
- 改完路径后，建议手动执行一次“同步数据”，确保新库里有完整数据。
- 如果当前路径由 `CODEX_USAGE_DATABASE` 控制，输入框会被锁定，应用内不能修改。

如果你想回到默认路径，点击“恢复默认路径”即可。

## 自动同步怎么用

概览页顶部可以设置同步频率。

可选项包括：

- 手动
- 10 秒
- 30 秒
- 1 分钟
- 5 分钟
- 10 分钟
- 15 分钟
- 30 分钟

建议：

- 日常观察数据时，用“手动”或“1 分钟”通常就够了
- 如果你正频繁使用 Codex，希望看板更快跟上变化，可以临时改成“10 秒”或“30 秒”

倒计时会显示距离下次自动同步还有多久。同步过程中，倒计时会切换成“同步中”。

## 常见问题

### 打开后没有数据

先检查两项：

1. `CODEX_HOME/sessions/` 下是否真的有 `.jsonl` 会话文件
2. 你是否已经点击过“同步数据”

如果仍然没有数据，进入“关于”页确认：

- Codex 目录是否正确
- SQLite 路径是否写到了你预期的位置

### 日期范围查不到结果

这通常表示该范围内没有已同步的数据。先确认：

1. 这个日期范围是否已经被同步进 SQLite
2. 你查询的日期是否超出可用数据范围

### 为什么路径输入框不能编辑

这通常表示当前数据库路径由环境变量 `CODEX_USAGE_DATABASE` 控制。此时应用只能读取该设置，不能在界面中覆盖它。

### 为什么同步后还有少量待同步

如果你正在使用 Codex，活跃的 session 文件仍会持续追加内容，所以同步刚结束时仍可能出现少量待同步会话。这是预期行为，不一定代表异常。
